# 源流直连 · 项目上下文
> CLAUDE.md · 最后更新：2026-06-28

## 项目定位
源流直连当前项目为“源流绿电直连自测工具”，是面向新能源项目开发者、EPC 方、用电企业能源管理人员的绿电直连项目可行性自测工具。工具通过五步流程录入现场踏勘数据，估算发电能力，匹配用户负荷，校验绿电直连合规指标，并生成可打印评估报告。

## 核心功能模块
- 基本信息：项目名称、地点、项目类型、能源形式、日期。
- 现场踏勘：屋顶光伏、地面光伏、风电、生物质、其他场地、电力设施、权属、用电客户、合规类检查、自定义踏勘项。
- 资源与负荷：发电能力汇总、12 时段用户负荷、Chart.js 发电-负荷匹配图、储能优化预览。
- 合规测算：基于发改能源〔2026〕688号第二条三项指标进行自动判定。
- 碳减排：年减排量、全生命周期减排量、等效植树量。
- 报告输出：结构化综合评估报告，支持浏览器打印或导出 PDF。

## 技术栈
- 形态：单文件 SPA。
- 前端：HTML、CSS、Vanilla JavaScript。
- 图表：Chart.js 3.9.1 CDN。
- 存储：无后端、无数据库、无 localStorage，用户数据仅在浏览器当前页面内计算。
- 导出：`window.print()` 与 `@media print`。
- 部署：GitHub Pages。

## 关键约束与风险
- 核心应用集中在 `index.html`，修改前必须控制范围，避免一次性大改导致回归困难。
- Chart.js 依赖 CDN，离线或微信环境网络异常时图表可能不可用。
- `calcRoof` 公式已调整为年有效发电小时直接输入，旧参数不可直接和新版横向对比。
- 合规算法仅依据公开指标，不构成投资建议或审批依据。
- 当前无后端，禁止引入数据上传或隐私风险，除非 Sam 明确要求产品形态升级。
- 微信内置浏览器、iOS Safari 的折叠交互、浮层定位、打印能力需要实测。

## 禁止事项
- 禁止引入后端、数据库、用户数据上传或持久化，除非 Sam 明确决策。
- 禁止修改政策指标阈值而不标明依据。
- 禁止把测算结果表述为审批结论或投资建议。
- 禁止未经验证声称微信内置浏览器兼容。
- 禁止顺手重构整个 `index.html` 或改变主流程。
